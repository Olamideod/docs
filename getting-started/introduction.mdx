# Welcome to protekt

Protekt is an authentication platform that gives you secure login, [multi-factor authentication (MFA)](/support/glossary#mfa-multi-factor-authentication), [single sign-on (SSO)](/support/glossary#sso-single-sign-on), and [role-based access control (RBAC)](/support/glossary#rbac-role-based-access-control) as part of a single system. 
## Core features

1. **User registration and login**: Email/password, social, and passwordless authentication
2. **Multi-factor authentication (MFA)**: SMS, authenticator apps, and biometric verification
3. **Session management**: Secure token handling and automatic refresh
4. **Role-based permissions**: Fine-grained access control for your application


## How it works

Your application redirects users to Protekt's hosted pages, Protekt handles authentication, then returns secure tokens to your callback URL.

**Example Workflow:**

```javascript server.js
// 1. Redirect to Protekt
const loginUrl = protekt.getLoginUrl({
  redirectUri: 'https://yourapp.com/callback'
});

// 2. Handle callback
const { user, tokens } = await protekt.handleCallback(code);

// 3. Validate requests
app.use('/api', protekt.validateJWT);
```

This [OAuth 2.0 flow](/support/glossary#oauth-and-flows) ensures your application never handles sensitive credentials directly. Protekt manages password storage, MFA challenges, and session security while you focus on your product logic.
