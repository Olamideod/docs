## Step 1: Create your application in protekt

Log into your [Protekt Dashboard](https://dashboard.protekt.com) and create a new application:

1. Click **Create Application**
2. Name it (e.g., "My App")
3. Set your callback URL: `http://localhost:3000/callback`
4. Copy your **Client ID** and **Client Secret**

The callback URL is where Protekt redirects users after they authenticate. During development, this points to your local machine. In production, you'll change this to your live domain.

## Step 2: Install this protekt sdk

To intall the proteck SDK, run the command:

```bash
npm install @protekt/node
```

The SDK handles all the OAuth protocol details, generating authorization URLs, exchanging codes for tokens, and validating responses. You could make raw HTTP requests to Protekt's API instead, but the SDK saves you from dealing with the complexity.

## Step 3: Configure protekt

Create a `protekt.js` file in your project:

```javascript protekt.js
const { Protekt } = require('@protekt/node');

const protekt = new Protekt({
  clientId: process.env.PROTEKT_CLIENT_ID,
  clientSecret: process.env.PROTEKT_CLIENT_SECRET,
  redirectUri: 'http://localhost:3000/callback'
});

module.exports = protekt;
```
<Tip>
Store your credentials in environment variables, not directly in code. Create a `.env` file:
</Tip>

```bash .env
PROTEKT_CLIENT_ID=your_client_id_here
PROTEKT_CLIENT_SECRET=your_client_secret_here
```
<Tip>
Add `.env` to your `.gitignore` so credentials never get committed to version control.
</Tip>

## Step 4: Add login route

Create a route that redirects users to Protekt's login page:

```javascript server.js
const express = require('express');
const protekt = require('./protekt');

const app = express();

app.get('/login', (req, res) => {
  const loginUrl = protekt.getLoginUrl();
  res.redirect(loginUrl);
});

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000');
});
```

When users visit `/login`, they're sent to Protekt's hosted authentication page. This page is fully managed, you don't need to build or maintain any login UI. Users can sign up, log in, reset passwords, or use social logins, all on Protekt's domain.


## Step 5: Handle the callback

After users authenticate, Protekt redirects them back to your callback URL with an authorization code. Exchange this code for access tokens:

```javascript server.js
app.get('/callback', async (req, res) => {
  const { code } = req.query;
  
  try {
    const { user, tokens } = await protekt.handleCallback(code);
    
    // Store user in session
    req.session.user = user;
    req.session.accessToken = tokens.accessToken;
    
    res.redirect('/dashboard');
  } catch (error) {
    console.error('Authentication failed:', error);
    res.redirect('/login?error=auth_failed');
  }
});
```

The authorization code in the URL is single-use and short-lived (valid for 60 seconds). You exchange it for tokens that your application uses to authenticate subsequent requests. This two-step process the code first, then tokens is part of the OAuth 2.0 Authorization Code flow, designed to keep tokens secure even if URLs get logged or shared.

## Step 6: Protect routes

Add middleware to verify users are authenticated before accessing protected routes:

```javascript server.js
function requireAuth(req, res, next) {
  if (!req.session.user) {
    return res.redirect('/login');
  }
  next();
}

app.get('/dashboard', requireAuth, (req, res) => {
  res.send(`Welcome, ${req.session.user.email}!`);
});
```

This middleware checks if a user exists in the session. If not, it redirects to login. If yes, it allows the request to proceed. You can apply this middleware to any route that should require authentication.

## Step 7: Add session management

Install and configure express-session to persist user sessions:

```bash
npm install express-session
```

```javascript server.js
const session = require('express-session');

app.use(session({
  secret: process.env.SESSION_SECRET,
  resave: false,
  saveUninitialized: false,
  cookie: { 
    secure: process.env.NODE_ENV === 'production',
    httpOnly: true,
    maxAge: 24 * 60 * 60 * 1000 // 24 hours
  }
}));
```

Sessions store user information server-side and give the browser a session ID cookie. The `httpOnly` flag prevents JavaScript from accessing the cookie, protecting against XSS attacks. The `secure` flag ensures cookies only transmit over HTTPS in production.

## Step 8: Test the flow

Start your server and test the authentication flow:

```bash
node server.js
```

1. Navigate to `http://localhost:3000/login`
2. You'll be redirected to Protekt's login page
3. Create an account or log in
4. You'll be redirected back to `/callback`, then to `/dashboard`
5. You should see "Welcome, [your-email]!"
