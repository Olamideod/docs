New releases and improvements to Protekt.

<Update label="January 15, 2025" tags={["New releases", "Improvements"]} rss={{ title: "January Updates", description: "Organization-level MFA requirements, token expiration changes, and SSO improvements" }}>
## New features

* **Organization-level MFA requirements**: Organizations can now require MFA for all members. Set in organization settings, and users without MFA are prompted to set it up on next login.
* **Token expiration defaults updated**: Default access token lifetime increased from 15 minutes to 60 minutes based on customer feedback. Refresh tokens remain valid for 30 days.

## Bug fixes and improvements

* Fixed rate limit headers: `X-RateLimit-Reset` now returns correct Unix timestamp (previously returned milliseconds instead of seconds)
* Fixed SSO callback timeout: SAML assertion processing now completes within 5 seconds (previously could timeout for assertions >100KB)
* Fixed user metadata updates: Updating metadata no longer overwrites unspecified fieldsâ€”only provided fields are updated

## SDK updates

* JavaScript SDK v2.1.0
* Python SDK v1.8.0  
* Go SDK v1.3.0
</Update>

<Update label="December 20, 2024" tags={["New releases", "Improvements"]} rss={{ title: "December Updates", description: "Magic link authentication, webhook filtering, and improved error responses" }}>
## Major releases

* **Magic link authentication**: Send passwordless login links via email. Users click the link to authenticate without entering passwords. [See guide](../guides/send-magic-link)
* **Webhook event filtering**: Filter webhooks by event type in dashboard settings to reduce unnecessary calls. Available events: `user.created`, `user.updated`, `user.deleted`, `login.success`, `login.failed`, `mfa.enabled`, `organization.created`

## API improvements

* **Enhanced error responses**: Error responses now include `error_code` for programmatic handling:
  ```json
  {
    "error": {
      "code": "invalid_email",
      "message": "Email address is invalid",
      "field": "email"
    }
  }
  ```

## Security updates

* Enhanced token validation: JWT signatures now validated against rotating keys, improving security without breaking existing integrations

## Bug fixes

* Fixed organization member listing pagination: `/organizations/{id}/members` endpoint now correctly returns `pagination` object with total count

## SDK updates

* JavaScript SDK v2.0.0
* Python SDK v1.7.0
* Go SDK v1.2.0
* React Native SDK v1.0.0 (new)
</Update>

<Update label="December 1, 2024" tags={["Breaking changes", "Improvements"]} rss={{ title: "December Updates", description: "Deprecated endpoint removal and session API enhancements" }}>
## Breaking changes

* **Removed deprecated `/auth/login` endpoint**: Direct login endpoint removed. Use OAuth flow instead:
  ```javascript
  // Old (removed)
  await fetch('https://api.protekt.com/auth/login', {
    method: 'POST',
    body: JSON.stringify({ email, password })
  });

  // New (use this)
  const loginUrl = protekt.getLoginUrl();
  res.redirect(loginUrl);
  ```
  [Migration guide](../guides/migrate-direct-login)

## Improvements

* **Session API enhancements**: `/sessions` endpoint now includes device information and last activity timestamp:
  ```json
  {
    "id": "sess_123",
    "userId": "usr_456",
    "device": "Chrome on macOS",
    "lastActivity": "2024-12-01T10:30:00Z",
    "createdAt": "2024-12-01T08:00:00Z"
  }
  ```

## SDK updates

* JavaScript SDK v1.9.0
* Python SDK v1.6.0
* Go SDK v1.1.0
</Update>

## Deprecation notices

### Scheduled for removal

**March 1, 2025: Legacy token format**

Tokens issued before December 2024 will no longer be accepted. Refresh all active sessions before this date.

**Action required:**
```javascript
// Force all users to re-authenticate
await protekt.sessions.revokeAll();
```

**April 1, 2025: API v0.x endpoints**

All v0.x API endpoints will return 410 Gone. Update to v1 endpoints.
