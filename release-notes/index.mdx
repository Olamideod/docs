## Release Schedule

We follow a predictable release schedule:

- **Major Releases**: Every 3 months (v1.0, v2.0, etc.)
- **Minor Releases**: Monthly (v1.1, v1.2, etc.)
- **Patch Releases**: Weekly as needed (v1.1.1, v1.1.2, etc.)

## Breaking Changes

We strive to maintain backward compatibility, but occasionally breaking changes are necessary. All breaking changes are:

1. **Announced in advance** via our blog and release notes
2. **Documented clearly** with migration guides
3. **Deprecated gradually** with clear timelines
4. **Supported during transition** with both old and new APIs
